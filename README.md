Проект представляет собой десктопное приложение на C# (Windows Forms), реализующее систему аутентификации пользователей и имитацию почтового сервиса. Особенностью архитектуры является отказ от классических СУБД в пользу  решения для данных на основе сериализации.

# Стек технологий
*   **UI:** Windows Forms
*   **Работа с данными:** Newtonsoft.Json (Json.NET)
*   **IO:** System.IO (File Management)
*   
## Архитектура и реализация данных
Вместо использования SQL-базы данных или хардкода учетных данных, в проекте реализована сериализация данных о пользователя в файле txt в формате json.

### Logic.cs
Центральный класс бизнес-логики, отвечающий за операции CRUD (Create, Read, Update, Delete) и авторизацию.

**Ключевые механизмы:**
1.  **Сериализация:** Данные пользователей и сообщений хранятся в виде типизированных списков `List<User>` и `List<Mail>`.
2.  **Персистентность:** При любом изменении данных (регистрация, отправка письма) происходит полная сериализация объектов в формат JSON и запись в локальные файлы `db.json` и `mail.json`.
3.  **Десериализация:** При запуске или запросе данных система вычитывает текстовые файлы и преобразует JSON обратно в объекты C#.

## Функциональность

*   **Регистрация пользователей:** Создание новой сущности `User` с полями (mail, login, password, role) и сохранение в файловое хранилище.
*   **Авторизация:** Проверка пары логин/пароль путем перебора десериализованной коллекции пользователей. Возвращает роль пользователя при успехе.
*   **Почтовая система:**
    *   Сохранение сообщений для конкретных пользователей.
    *   Фильтрация и вывод сообщений, адресованных текущему авторизованному пользователю (`login_user`).
*   **Администрирование:** Вывод списков всех зарегистрированных логинов и глобальной истории переписки.
